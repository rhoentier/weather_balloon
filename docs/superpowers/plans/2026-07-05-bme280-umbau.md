# BMP280 → BME280 Umbau Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Den bisher dokumentierten Platzhalter-Sensor BMP280 im gesamten Projekt durch den tatsächlich verbauten BME280 ersetzen (inkl. Luftfeuchtigkeit) und ihn erstmals real per I²C am Heltec V2 auslesen.

**Architecture:** `lib/telemetry/bmp280` (eigene, nativ getestete Kompensationsformel) wird komplett gelöscht. Für den BME280 wird bewusst die fertige Adafruit-Bibliothek genutzt (I²C + Kompensation intern) — ein neuer, dünner Wrapper `src/flight/bme_sensor` ruft sie auf. `TelemetryRecord` bekommt `has_bme` (umbenannt aus `has_bmp`) + neues Feld `humidity_pct`; CSV wächst auf 12 Spalten. `DisplayState.bmp_ok` → `bme_ok`.

**Tech Stack:** C++17 (PlatformIO `native`-Env, Unity), Arduino-Framework + Adafruit BME280 Library (PlatformIO `flight`-Env).

**Spec:** `docs/superpowers/specs/2026-07-05-bme280-umbau-design.md`

## Global Constraints

- Sprache im Projekt: **Deutsch** (Kommentare, Commits, Doku).
- **Hardware-frei bleibt hardware-frei:** `lib/telemetry/` darf weiterhin kein `<Arduino.h>` einbinden. Der neue `bme_sensor`-Wrapper (Arduino + Adafruit-Lib) lebt in `src/flight/`, NICHT in `lib/`.
- **TDD für alles in `lib/telemetry/`:** erst Test, RED sehen, dann GREEN. Der `src/flight`-Wrapper wird nicht nativ getestet (kein Unity-Test dafür — analog zu `oled.cpp`/`sd_log.cpp`), sondern am Board verifiziert (späterer, eigener Schritt außerhalb dieses Plans).
- **YAGNI:** QNH fest `1013.25f` hPa, I²C-Adresse fest `0x76`, kein eigener Lesetakt (synchron zum GPS-GGA-Takt), kein I²C-Adress-Scan.
- **Namespace:** `TelemetryRecord`/`DisplayState`-Änderungen bleiben in `namespace telemetry`.
- **Test-Konvention:** ein Ordner pro Baustein unter `test/`, eigenes `main()` mit `UNITY_BEGIN/END`.
- **Kein Git-Repo:** Dieses Projekt ist (Stand jetzt) nicht unter Git-Versionierung. Die „Commit"-Schritte sind **optional** — falls `git rev-parse` fehlschlägt, überspringen und den Task trotzdem als abgeschlossen markieren.
- **Build/Test-Kommando:** `export PATH="$PATH:$HOME/.platformio/penv/bin"` einmalig, dann `pio test -e native` (lib-Tests) bzw. `pio run -e flight` (Compile-Check des Board-Codes, kein Board nötig).
- **Software-Flow-Doku mitpflegen** (CLAUDE.md-Vorgabe): `docs/software-flow.md` wird als letzter Task aktualisiert.

```cpp
// bme_sensor.h — BME280 (Temperatur/Druck/Höhe/Feuchte) via Adafruit-Bibliothek.
//
// Board-Hardware-Teil (kein nativer Test, analog zu oled.cpp/sd_log.cpp) —
// wird am Heltec V2 verifiziert. I2C-Adresse fest 0x76 (typischer Default
// günstiger BME280-Breakouts). QNH fest 1013,25 hPa (Standardatmosphäre,
// keine Boden-Kalibrierung — YAGNI): Höhe ist damit eine RELATIVE Größe,
// für die Flugphasen-Erkennung reicht die Änderung, nicht der Absolutwert.

#ifndef FLIGHT_BME_SENSOR_H
#define FLIGHT_BME_SENSOR_H

#include "record.h"

// I2C muss vorher laufen (z.B. über oled_begin(), das denselben Bus startet).
// Rückgabe: true, wenn der Sensor am Bus antwortet (Chip-ID ok).
bool bme_begin();

// Liest den Sensor und füllt has_bme/temp_c/pressure_hpa/alt_baro_m/
// humidity_pct in rec. Ohne erfolgreichen bme_begin() vorher bleibt rec
// unverändert (has_bme bleibt, was es vorher war).
void bme_read(telemetry::TelemetryRecord& rec);

#endif // FLIGHT_BME_SENSOR_H
```

- [ ] **Step 2: Implementierung schreiben**

Create `src/flight/bme_sensor.cpp`:

```cpp
// bme_sensor.cpp — siehe Header. Adafruit_BME280, HW-I2C, Adresse 0x76.
#include "bme_sensor.h"
#include <Adafruit_BME280.h>

namespace {
constexpr uint8_t  kBmeAddress   = 0x76;
constexpr float    kSeaLevelHpa  = 1013.25f;  // Standardatmosphäre (YAGNI)

Adafruit_BME280 g_bme;
bool             g_bme_ok = false;
}  // namespace

bool bme_begin() {
    g_bme_ok = g_bme.begin(kBmeAddress);
    return g_bme_ok;
}

void bme_read(telemetry::TelemetryRecord& rec) {
    if (!g_bme_ok) return;

    rec.has_bme      = true;
    rec.temp_c       = g_bme.readTemperature();
    rec.pressure_hpa = g_bme.readPressure() / 100.0f;   // Pa -> hPa
    rec.alt_baro_m   = g_bme.readAltitude(kSeaLevelHpa);
    rec.humidity_pct = g_bme.readHumidity();
}
```
